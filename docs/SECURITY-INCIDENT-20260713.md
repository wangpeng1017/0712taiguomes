# 阿里云 ECS 安全事件记录（2026-07-13）

## 事件摘要

服务器 `8.130.182.148` 发现两个以 root 身份运行的异常 ELF：`/10eEJz`、`/wZevxL`，并存在 `/g0` 配置及 `/dev/shm/let` 残留。异常进程主动连接多个境外 IP 的 53/80/123/443 端口。

`/etc/rc.d/rc.local` 被加入远程下载执行入口：`178.16.52.253/1utih`。这按 root 级入侵处理。

## 已执行处置

- 保存进程、网络连接、SHA-256、恶意文件和被修改启动脚本到 `/root/security-quarantine-20260713-221623`
- 终止异常进程，删除原始路径，隔离样本权限设为 `000`
- 删除 `rc.local` 恶意下载入口并复查未复活
- 隔离 `/dev/shm/let` 和针对不存在用户 `sysupdate` 的隐藏 sudo 规则
- 停止并禁用持续失败重启的 `lims.service`、`mihomo.service`
- 停止并禁用无数据库/无连接的 PostgreSQL、空 Redis，以及 3999 临时 Node 服务
- journal 从约 4GB 压缩至约 456MB，清理 npm/pip/pnpm 开发缓存
- 系统盘使用率由 89% 降至 79%，部署后约 82%
- root SSH 从旧 RSA 密钥轮换为独立 Ed25519 密钥，旧密钥已撤销，密码登录保持关闭

## 验证结果

- 异常进程未复活，原始文件路径不存在
- 保留的 LIMS、NPI、AgentBase、启梦、点云、宇树服务健康检查正常
- 泰国 MES 使用独立 MySQL 数据库账号，应用仅监听本机端口

## 剩余风险

在线清理无法证明内核、系统二进制、历史凭据和所有业务密钥未被篡改或泄露。建议尽快：

1. 新建干净 ECS，按最小暴露面重新部署业务。
2. 轮换所有数据库账号、第三方 API Key、Git 凭据和其他业务密钥。
3. MySQL 3308/3309 及 Node 业务端口限制为可信来源或仅本机访问。
4. 对公开 Node/Next 服务做依赖漏洞扫描和入口审计。
5. ECS 重建前持续监控异常进程、外连和启动项变化。

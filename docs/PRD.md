## 一、项目概述

### 1.1 项目愿景

工单驱动生产，批次支撑追溯，模具关联寿命，日报自动汇总——用最小成本实现泰国工厂汽车电子结构件（注塑+冲压，线束预留）的小批次可追溯生产管理，不做重型 MES/APS/WMS/QMS。

### 1.2 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 系统管理员 | 基础数据/权限/编码规则维护 | 一次配置，长期稳定 |
| 生产计划员 | 创建/下达工单，看进度和日报 | 工单状态一目了然 |
| 班组长 | 开工确认、设备/模具确认、数量审核 | 现场操作简单快 |
| 操作员 | 扫码/选工单，录入良品/不良/退料 | 表单字段少、不出错 |
| 仓库人员 | 原材料入库/领退料/半成品成品入库/不良隔离 | 数量闭环不能对不上 |
| 模具管理员 | 模具台账/保养/维修/状态维护 | 寿命预警提前看到 |
| 质量人员 | 不良确认、原因维护、追溯查询 | 一查到底不用问人 |
| 管理人员 | 日报/汇总/模具寿命/良率/追溯结果 | 数据自动汇总不用人工凑 Excel |

### 1.3 核心业务闭环

```
原材料入库 → 工单创建/下达 → 领料 → 开工(绑设备/模具/物料批次) → 生产报工(良品/不良)
  → 模具模次/冲次累计 → 良品入库 → 自动生成日报 → 批次追溯查询
```

关键设计取舍（对应王老师原始 9 条诉求）：
- **小批次追溯，非单件追溯**：追溯单元 = 工单+日期+班次+SKU+设备+模具+原材料批次，换任一维度即拆新批次
- **模具寿命=累计次数/设计寿命**，报工自动累加，80% 预警/100% 强制主管确认
- **数量闭环**：总生产=良品+不良；实际消耗=领料-退料；入库数量≤良品数量
- **一期不做**：单件二维码、PLC自动采集、APS排产、完整WMS/QMS、ERP深度集成、设备OEE

## 三、功能详情（仅摘要，完整验收标准见 SPEC v0.1 §16）

### F002: 生产工单管理
- **用户故事**: 作为生产计划员，我希望创建工单并下达到现场，让操作员能看到并开工
- **验收标准**:
  - [x] 工单状态机：未下达→已下达→生产中→暂停/已完工→已关闭，服务端校验合法转换
  - [x] 未下达、暂停、已完工、已关闭工单不可报工
  - [x] 累计良品达到计划数量后自动完工
  - [x] 工单进度实时统计：计划数量/累计良品/累计不良/完成率/当前批次数
  - [x] 支持新增、详情、受控编辑、状态流转和未下达无引用工单删除

### F003/F004: 注塑/冲压生产报工
- **验收标准**:
  - [x] 菜单默认进入生产批次列表，新增报工通过右上角按钮打开抽屉表单
  - [x] 列表支持关键词/状态筛选、批次详情和安全作废
  - [x] 已入库批次禁止作废；作废时回滚物料库存、模具次数和工单进度
  - [x] 模具状态=维修中/停用/报废 → 开工时禁用并由服务端拒绝
  - [x] 模具寿命使用率≥100% → 开工需二次确认并记录主管姓名/确认时间
  - [x] 批次号按规则自动生成：`{INJ|STP}-{YYYYMMDD}-{D|N}-{设备编号}-{流水号3位}`
  - [x] 报工提交后：总生产=良品+不良、材料利用率、不良率、（冲压）冲次=总生产/单次出件数 全部自动计算
  - [x] 提交后模具 currentCount 自动累加，跨越预警/强制阈值时提示
  - [x] 服务端校验工单、设备、模具、SKU、物料类型与状态的适配关系
  - [x] 领料、退料、消耗、废料和不良明细执行数量闭环校验

### F006: 模具寿命与保养管理
- **验收标准**:
  - [x] 寿命使用率=当前累计次数/设计寿命，实时展示进度条+状态色
  - [x] 登记保养后模具状态恢复"可用"，记录保养时间/人员/内容/更换部件
  - [x] 模具支持新增、编辑、状态维护和无引用安全删除；保养记录支持编辑和删除纠错

### F007: 批次追溯查询
- **验收标准**:
  - [x] 正向：搜索物料批次号 → 展示所有使用该批次的工单/生产批次/良品/不良/入库
  - [x] 反向：搜索生产批次号 → 展示工单/设备/模具/物料批次/班次/操作员/不良记录/入库记录/超寿命确认全链路
  - [x] 模具追溯：搜索模具编号 → 历史生产批次/累计模次/保养记录/维修记录

### F008: 每日生产日报
- **验收标准**:
  - [x] 按曼谷自然日实时生成，来源于生产报工+入库+不良+模具使用记录
  - [x] 支持在选定日报日期内按 8 个维度切换汇总（日期/班次/工单/SKU/设备/模具/不良原因/物料批次）
  - [x] 支持导出（Demo 用 CSV 代替 Excel，字段一致）

### 通用 CRUD 与审计规则

- 主数据：支持新增、编辑、启停；未被业务引用的数据可硬删除，已引用数据必须停用
- 工单：未生效数据可删除；进入生产链路后保留记录，仅允许受控编辑和状态流转
- 生产批次：不直接编辑已生效数量，采用“作废并重新报工”；作废执行库存、模具和工单联动回滚
- 物料：批次元数据可编辑，未使用批次可删除；入库与不良隔离记录支持编辑、撤销
- 报表、追溯和仪表盘是聚合查询页面，不提供破坏来源数据的增删改入口

### 中英文界面

- 顶栏提供中文 / English 分段切换，并在浏览器中持久化选择
- 导航、页面标题、表格、表单、弹窗、状态、提示以及 Ant Design 基础控件同步切换
- 用户维护的产品名、模具名等业务数据保持原始录入值，不自动篡改
- 顶栏不再显示 Demo 锚点日期，左上角品牌统一简化为 `MES`

## 四、数据模型概览

详见 `prisma/schema.prisma`（唯一权威）。核心实体对应 SPEC §10：

| 实体 | 说明 |
|------|------|
| ProductSku / MaterialMaster / EquipmentMaster / MoldMaster | 主数据 |
| WorkOrder | 工单，1:N ProductionBatch |
| MaterialLot / MaterialIssue / MaterialReturn | 物料批次入库/领料/退料 |
| ProductionBatch | 生产批次（注塑/冲压报工的核心记录），N:1 WorkOrder/EquipmentMaster/MoldMaster |
| DefectRecord | 不良记录，1:N per Batch |
| StockInRecord | 半成品/成品入库、不良隔离 |
| MoldMaintenanceRecord | 模具保养/维修记录 |
